# Cloud Concepts

## Value of Cloud Computing

Cloud computing replaces large upfront infrastructure purchases with on-demand service consumption. It increases speed, supports experimentation, and shifts capacity planning from hardware procurement to service configuration.

## Benefits of the AWS Cloud

- Trade capital expense for variable expense
- Benefit from economies of scale
- Stop guessing capacity requirements
- Increase speed and agility
- Reduce time spent operating physical data centers
- Go global in minutes

## Shared Responsibility Model

AWS is responsible for security of the cloud, including the facilities, hardware, and managed infrastructure that run AWS services.

Customers are responsible for security in the cloud, including identity configuration, data protection, operating system administration, network controls, and workload settings based on the service model.

## High Availability and Elasticity

Availability means workloads remain accessible despite failures. Elasticity means resources can scale up or down as demand changes. These characteristics are easier to achieve in cloud environments because infrastructure is provisioned through services rather than physical procurement.

## Consumption Model

AWS services are generally paid for based on usage. This supports short-lived environments, experimentation, and tighter alignment between cost and business demand.
