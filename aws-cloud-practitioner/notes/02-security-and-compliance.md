# Security and Compliance

## IAM Fundamentals

IAM controls authentication and authorization in AWS. Identities include users, groups, roles, and policies. Permissions should be assigned according to least privilege.

## Root User Protection

The root user has unrestricted account access. It should be protected with a strong password and MFA, and it should not be used for routine administration.

## MFA and Strong Access Controls

Multi-factor authentication adds an extra verification step beyond a password. It reduces the risk associated with compromised credentials and is a standard control for privileged access.

## Data Protection

AWS supports encryption in transit and at rest across many services. Customers must choose appropriate encryption settings, protect keys, and manage access to sensitive data.

## Compliance and Governance

AWS provides compliance programs and security services, but customers remain responsible for configuring workloads in line with their own regulatory and organizational requirements. Logging, access review, and policy enforcement are key governance practices.
