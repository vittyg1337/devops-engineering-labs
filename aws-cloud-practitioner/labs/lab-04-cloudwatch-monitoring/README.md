# Lab 04 - CloudWatch Monitoring

## Objective

Review how Amazon CloudWatch provides metrics, logs, and alarms for AWS resource visibility and basic operational monitoring.

## Services Used

- Amazon CloudWatch
- Amazon EC2

## What I Built

- Navigated CloudWatch metrics for an AWS resource
- Reviewed standard monitoring data
- Planned an alarm for a simple operational threshold

## Key Concepts

CloudWatch is the native AWS monitoring service for metrics, logs, alarms, and dashboards. It helps operators observe performance and react to operational events.

Metrics show numerical performance data over time. Alarms evaluate those metrics and can trigger actions when thresholds are crossed.

## Steps Performed

1. Open CloudWatch and review available metrics.
2. Select an AWS resource such as an EC2 instance.
3. Inspect default monitoring data points.
4. Create or outline an alarm threshold.
5. Review how monitoring supports operational response.
