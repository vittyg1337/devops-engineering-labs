# Lab 03 - EC2 Linux Basics

## Objective

Launch a basic Amazon EC2 Linux instance and become familiar with core compute concepts, instance access, and common operating system tasks.

## Services Used

- Amazon EC2
- Amazon VPC
- Security Groups

## What I Built

- Launched a Linux EC2 instance
- Reviewed instance type, AMI, and key pair selection
- Connected to the instance for basic command-line interaction
- Observed how security groups control inbound access

## Key Concepts

Amazon EC2 provides resizable virtual servers in the cloud. An instance launch combines an Amazon Machine Image, instance type, network placement, storage, and access configuration.

Security groups act as virtual firewalls for instances. They define which inbound and outbound traffic is allowed.

## Steps Performed

1. Select an Amazon Linux AMI.
2. Choose a small instance type suitable for foundational practice.
3. Configure networking and attach a security group.
4. Launch the instance with a key pair.
5. Connect and run basic Linux commands.
6. Stop and terminate the instance after validation.
